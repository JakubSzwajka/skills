# Weekly Sales Review

Use this for one product or across the portfolio.

## Inputs

For each product:

- current product state
- current ICP hypothesis
- active proof assets
- outreach/conversation count
- activation and retention evidence
- payment or pricing signals
- top objections
- current build work

## Review Questions

1. Who did we talk to this week?
2. Which segment showed the strongest pain?
3. What did they already try or use?
4. Where did they stall before first value?
5. What created trust?
6. What broke trust?
7. Did anyone ask for the next step, another use, payment, or referral?
8. Which objection repeated?
9. Which current build task is directly tied to sales evidence?
10. Which build task is avoidance?

## Output Format

```txt
Project: <name>
Commercial stage: <idea/building/live/no customers/first users/first paid/repeatable>
This week's evidence: <short bullets>
Current bottleneck: <acquisition | activation | trust | retention | payment | positioning | product>
Next sales action: <one action>
Next build action: <only if tied to sales evidence>
Stop doing: <one thing>
Decision needed: <if any>
```

## Portfolio Allocation Rule

When multiple products compete for time:

- prioritize the product with the clearest buyer access and fastest learning loop
- do not reward the project that merely has the most comfortable build work
- keep one primary sales experiment active at a time unless there is strong evidence to parallelize
- if a product cannot be sold because it is not usable, define the smallest build needed to run sales discovery
