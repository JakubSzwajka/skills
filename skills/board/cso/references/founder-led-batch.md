# Founder-Led Batch

A founder-led batch is a small, inspectable sales-learning sprint. It is not a launch.

## When To Use

- The product has a concrete proof asset or usable workflow.
- The team needs real user signal.
- Every response can be personally reviewed.
- The goal is learning, activation, and retention evidence.

Do not use a batch as a substitute for broad launch readiness.

## Default Batch Shape

Duration: 5-10 business days.

Inputs:

- one buyer segment
- one concrete job-to-be-done
- one proof asset or product path
- one primary ask
- one tracking table

Default size:

- 5 existing users/testers or warm contacts
- 10 friends-of-friends or trusted adjacent contacts
- 20 manually selected prospects
- 1-3 founder posts only if they support the same hypothesis
- one follow-up maximum unless the person engages

## Tracker Columns

- name
- profile/source URL
- lead source
- segment
- suspected painful workflow
- why they might care
- first message date
- follow-up date
- reply status
- fit verdict
- topic/problem shared
- product path started
- first value reached
- useful signal captured
- noisy/missing/trust issue captured
- second value or retention signal
- payment/upgrade/referral/objection
- next action

## Daily Rhythm

Day 0:

- choose segment and hypothesis
- prepare proof asset
- prepare tracker
- draft messages
- define stop/change rules

Days 1-2:

- contact warm/high-fit people
- publish only if useful to the same hypothesis
- log every response

Days 3-5:

- run discovery/setup calls
- push best-fit people to first value
- classify objections
- adjust message only from evidence

Days 6-10:

- ask whether they want the next output/action
- ask for payment, intro, or clear refusal where appropriate
- decide repeat, narrow, change segment, or pause

## Success Signals

Minimum:

- 20-30 people contacted
- 5 replies or real conversations
- 3 concrete pain/workflow submissions
- 2 product paths started
- 1 specific useful signal
- 1 concrete objection that changes product/copy/sales thinking

Strong:

- several users reach second value
- one payment, upgrade, referral, or serious buying question
- repeated language from buyers that sharpens positioning

Pause if:

- people cannot name a concrete pain
- first value does not happen
- repeated trust objections lack a real answer
- founder labor is silently doing the product's work
- legal, pricing, or promise language is inconsistent
